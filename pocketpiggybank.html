Pocket Piggy Bank

Pocket Piggy Bank is a mobile and web app that makes saving up for that next big purchase, adventure, or life event easy. We help users manage their finances in a stress-free and informative environment. We’re dedicated to helping each and everyone of our users build best practices for managing their finances without being financial experts. We pride ourselves on being Trustworthy, Accessible and Stress-Free. Users can trust that we’ll give them the best money-saving guidance, while helping them have a better understanding of wealth management. We’ll do the hard work so users can sit back, relax and get excited for that upcoming purchase!

My Process

Brand Guidelines

In order to ensure cohesive design across all screens, I created brand guidelines. These guidelines include logo design and usage, as well as brand colors, typography, imagery and writing style.

User Flow Diagram

I defined three user stories in order to build a user flow diagram to map out how users will navigate through the product:

  • As a new user, I need to be able to create a profile, so that the financial advice I receive is personalized to me, my spending habits, and tailored to my goals.
  • As a user, I want to receive a personalized savings plan, so that I can save enough money to reach my goal in time.
  • As a user, I need to be able to tell the tool what my savings goal is and how long I have to reach it, so that I can save accordingly.

Low and Mid-Fidelity Wireframes

Having defined my user flow diagram, I designed the screens for one of my user stories: As a user, I need to be able to tell the tool what my savings goal is and how long I have to reach it, so that I can save accordingly. I started with low-fidelity frames, then mid-fidelity frames which I used to create a prototype and conduct user testing.

User Testing

I asked my testers to set a new goal for an upcoming trip and gathered their feedback to make changes before designing my high-fidelity wireframes.

Prototype

High-Fidelity Wireframes and Mockups

After testing my prototype on users I was able to determine what changes needed to be made when designing my final wireframes. I also designed a second breakpoint, a desktop browser, for one of my screens.
